# DHGLMs analysis — README

This folder contains R analysis scripts, data CSVs, and saved model objects used for the community/individual/population-level DHGLM analyses for the JAE revision.

**Contents**

*Folder: analysis*
- `community_level_analysis.R`: R script that performs the community-level analysis. It typically loads `functions.R`, reads the community data CSV, fits or loads a model (using `brms` in the current workflow), creates figures, and saves results. Run interactively (source in RStudio) or with `Rscript` after ensuring required packages are installed.

- `community_level_data.csv`: Tabular data (CSV) used by `community_level_analysis.R`. Contains the processed community-level variables used as input for modeling and plotting.

- `community_level_model.rds`: Saved R object (created with `saveRDS()`), typically a `brmsfit` or other fitted model used by `community_level_analysis.R`. Load with `readRDS('community_level_model.rds')` to inspect or generate predictions.

- `individual_level_analysis.R`: R script that performs the individual-level analysis. Structure and usage are analogous to `community_level_analysis.R` but operate at the individual scale. It may produce plots and saved model objects.

*Folder: analysis*

- `individual_level_data.csv`: CSV file with the individual-level dataset used by `individual_level_analysis.R`.

- `individual_level_model.rds`: Saved model object for the individual-level analysis. Load with `readRDS()`.

- `population_level_analysis.R`: R script for population-level analyses. Similar pattern: load helper functions from `functions.R`, read `population_level_data.csv`, run or load model, produce figures and summaries.

- `population_level_data.csv`: CSV file with population-level data used by `population_level_analysis.R`.

- `population_level_model.rds`: Saved model object for the population-level analysis.

- `functions.R`: Utility functions used by the analysis scripts. This file contains reusable code (data transforms, plotting helpers, helper wrappers for model summaries/plots). Source this file before running the analysis scripts (scripts typically call `source('functions.R')`).

**Usage / Quick start**
1. Open an R session (R >= 4.0 recommended; current development used R 4.5.2).
2. Install required packages if needed (examples used in this repo include: `tidyverse`/`dplyr`, `ggplot2`, `ggridges`, `patchwork` or `gridExtra`, `brms` and its dependencies).
3. In R or RStudio:

```r
source('functions.R')
source('community_level_analysis.R')    # run community-level analysis
source('individual_level_analysis.R')   # run individual-level analysis
source('population_level_analysis.R')   # run population-level analysis
```

4. To load a saved model directly:

```r
m <- readRDS('models/community_level_model.rds')
# then inspect or predict with m
```

**Notes & provenance**
- The `.rds` model files were generated by the analysis scripts; they are binary R objects. Do not edit them by hand.
- Plots in the current workflow use `ggplot2` and in-session usage observed `ggridges` (density ridges) and `patchwork` or `gridExtra` for arrangement.
- If you only need to generate figures and summaries from existing models, load the `.rds` files and run the plotting/summary sections of the analysis scripts rather than refitting.
